# 阶段8：指标设计 & ROI测算

> 不能衡量的就不能管理。本阶段目标：**设计可衡量、可归因的指标体系，测算产品的投入产出比，用数据驱动决策**。

---

## AI 执行指引

进入本阶段时，按以下步骤操作：

1. **读取前置产物**：读取功能设计文档、商业模式设计文档
2. **逐步引导提问**：按下方「引导提问」分3个部分依次讨论
3. **确保指标可归因**：每个驱动指标都必须能回答"这个指标变差了，该找谁"
4. **生成指标设计文档和ROI测算表**：按「输出模板」生成
5. **触发审核**：加载 `references/reviewers/reviewer-finance.md`，以财务视角审核
6. **归档**：将文档保存到 `~/.workbuddy/products/{产品名称}/08-metrics/`
7. **下一步**：进入 **阶段9：产物输出**

## 本阶段目标

输出：
- **指标体系设计**（北极星 + 驱动 + 健康）

## 引导提问

### 第一部分：指标设计

1. **北极星指标**——产品最重要的一个指标

   > 好的北极星指标：反映核心价值、可衡量、可归因、可行动。

   | 候选指标 | 定义 | 优点 | 缺点 | 你的选择 |
   |---------|------|------|------|---------|
   | | | | | |

   **检验**：
   - 这个指标变好了，用户体验一定变好了吗？
   - 这个指标能被团队日常行动影响吗？
   - 有没有"虚荣指标"的风险？（数字好看但不反映真实价值）

2. **驱动指标**——拆解北极星的杠杆点

   > 每个驱动指标必须能回答：这个指标变差了，该做什么？

   | 驱动指标 | 定义 | 归因方向 | 负责团队 | 当前值 | 目标值 |
   |---------|------|---------|---------|--------|--------|
   | | | | | | |

   **归因方向示例**：
   - 入池量下降 → 上游拓展不足
   - 转化率下降 → 产品体验问题
   - 留存率下降 → 核心价值不足

3. **健康指标**——监测产品健康度的"报警器"

   | 健康指标 | 定义 | 预警阈值 | 超阈值后的行动 |
   |---------|------|---------|--------------|
   | | | | |

   > 预警阈值必须有行动对应。没有对应行动的阈值只是数字。

4. **指标目标值设定**

   | 指标 | 基准来源 | 当前值 | 3个月目标 | 6个月目标 | 1年目标 |
   |------|---------|--------|----------|----------|--------|
   | | 行业/历史/竞品 | | | | |

### 第二部分：数据埋点设计

   | | | | |

6. **每个事件的关键字段**

   | 事件 | 字段名 | 字段说明 | 类型 | 示例值 |
   |------|--------|---------|------|-------|
   | | | | String/Number/Enum/DateTime | |

   > 必须包含：用户ID、时间戳、来源渠道。

7. **埋点质量保障**

   | 保障措施 | 说明 |
   |---------|------|
   | 埋点测试 | 上线前验证埋点正确性 |
   | 数据校验 | 上线后监控数据完整性 |
   | 异常告警 | 数据量突降/突增自动告警 |

## 虚拟角色审核：财务视角

指标设计和ROI测算完成后，加载 `references/reviewers/reviewer-finance.md`，以财务视角审核，重点：

- ROI测算是否合理？有没有过于乐观？
- 投入预估是否完整？有没有漏算？
- 产出预测依据是什么？有没有吹牛？
- 回本周期是否可接受？
- 敏感性分析是否覆盖了最悲观情况？

审核后输出 3-5 个关键问题和修改建议，由用户确认后再进入下一阶段。
