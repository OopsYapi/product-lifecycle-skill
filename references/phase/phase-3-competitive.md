# 阶段3：竞品分析

> 竞品分析的目标不是"抄竞品"，而是：**理解竞品为什么这么做，找到差异化定位，避免重复造轮子**。

---

## AI 执行指引

进入本阶段时，按以下步骤操作：

1. **读取前置产物**：读取 `02-solutions/` 下的方案设计文档，了解产品定位和核心功能
2. **协助识别竞品**：用 WebSearch 搜索相关竞品，帮用户补全竞品列表
3. **逐步引导提问**：按下方「引导提问」分4个部分依次讨论
4. **生成竞品分析报告**：按「输出模板」生成
5. **触发审核**：加载 `references/reviewers/reviewer-marketing.md`，以市场部视角审核
6. **归档**：将报告保存到 `~/.workbuddy/products/{产品名称}/03-competitive/{产品名称}-竞品分析.md`
7. **判断下一步**：进入 **阶段4：商业模式 & 盈利设计**

---

## 本阶段目标

输出 **竞品分析报告**，包含：
- 竞品全景图（直接/间接/替代）
- 多维度对比（功能/定价/体验/商业模式/市场策略）
- 差异化机会识别
- 对产品设计的启示

---

## 引导提问

### 第一部分：识别竞品

1. **竞品不一定是"做一样事的产品"，而是"用户解决同样问题的任何方案"**

   | 竞品类型 | 说明 | 你的竞品 |
   |---------|------|---------|
   | **直接竞品** | 做一模一样的事 | |
   | **间接竞品** | 解决同样问题，方式不同 | |
   | **替代方案** | 用户用完全不同的方式解决问题 | |
   | **潜在进入者** | 有能力做但还没做的玩家 | |

2. **每个竞品的基本信息**

   | 竞品名称 | 所属公司 | 成立时间 | 融资阶段 | 用户规模 | 核心定位 |
   |---------|---------|---------|---------|---------|---------|
   | | | | | | |

### 第二部分：多维度对比

   | 定价维度 | 我们的方案 | 竞品A | 竞品B |

   | 定价模式 | | | |

5. **用户体验对比**——如果有条件，亲自用一下竞品

   | 体验维度 | 我们的方案 | 竞品A | 竞品B |

### 第三部分：优势/劣势分析

8. **竞品的核心优势和劣势**

   | 竞品 | 核心优势 | 优势来源 | 核心劣势 | 劣势原因 |
   |------|---------|---------|---------|---------|
   | 竞品A | | | | |
   | 竞品B | | | | |

   > 优势来源：先发优势 / 技术壁垒 / 资源壁垒 / 网络效应 / 品牌认知 / 政策牌照

### 第四部分：差异化机会

9. **三个关键问题**

   - 竞品没有覆盖的用户需求/场景有哪些？→ **这是我们的机会**
   - 我们可以做得比竞品更好的地方？→ **这是我们的差异化**
   - 竞品做错了/做得不好的地方？→ **这是我们的切入点**

## 虚拟角色审核：市场部视角

竞品分析报告完成后，加载 `references/reviewers/reviewer-marketing.md`，以市场部视角审核，重点：

- 竞品识别是否全面？有没有遗漏重要竞品？
- 竞品分析是否客观？有没有高估/低估竞品？
- 差异化机会是否真实？还是"自以为差异化"？
- 市场进入策略是否可行？

审核后输出 3-5 个关键问题和修改建议，由用户确认后再进入下一阶段。

- 进入 **阶段4：商业模式 & 盈利设计**
