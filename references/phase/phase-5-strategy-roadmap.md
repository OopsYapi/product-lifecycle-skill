# 阶段5：战略规划 & 路线图

> 没有战略的产品是"功能堆砌"，没有路线图的战略是"空中楼阁"。本阶段目标：**明确产品长期方向，以及短期内先做什么、后做什么**。

---

## AI 执行指引

进入本阶段时，按以下步骤操作：

1. **读取前置产物**：读取方案设计文档、竞品分析报告、商业模式设计文档
2. **逐步引导提问**：按下方「引导提问」分3个部分依次讨论
3. **确保战略和前面阶段一致**：愿景不能和需求/方案/商业模式矛盾
4. **生成产品战略规划 & 路线图**：按「输出模板」生成
5. **触发审核**：加载 `references/reviewers/reviewer-ceo.md` 和 `references/reviewers/reviewer-marketing.md`，分别以CEO视角和市场部视角审核
6. **归档**：将文档保存到 `~/.workbuddy/products/{产品名称}/05-strategy/{产品名称}-战略规划.md`
7. **判断下一步**：
   - 平台型产品 → 进入 **阶段6：平台架构 & 开放API**
   - 非平台型产品 → 直接进入 **阶段7：功能设计 & 异常场景**

## 引导提问

### 第一部分：产品战略

1. **3年愿景**——3年后，这个产品在用户心中是什么位置？

   > 好的愿景有三个特征：具体、可衡量、有野心但不离谱。

   **格式参考**：
   - 3年后，成为 [领域] 的 [定位]
   - 服务 [数量] 家客户，覆盖 [范围]
   - 核心指标达到 [具体数字]

2. **核心价值主张**——用一句话说清楚：我们为什么存在？

   > 帮助 [目标用户] 解决 [核心问题]，通过 [核心能力]，实现 [核心价值]

3. **产品边界**——同样重要的是"我们不做什么"

   - 不服务哪些用户？
   - 不解决哪些问题？
   - 不进入哪些市场？
   - 不用什么方式赚钱？

   > 边界越清晰，产品越聚焦。

4. **竞争策略**

   | 策略 | 说明 | 适用条件 | 你的选择 |
   |------|------|---------|---------|
   | 成本领先 | 最低价格 | 有成本优势 | |
   | 差异化 | 功能/体验最好 | 有差异化能力 | |
   | 专注细分 | 聚焦某个细分市场 | 细分市场足够大 | |
   | 生态绑定 | 绑定大生态 | 有生态合作伙伴 | |
   | 平台化 | 做基础设施 | 有双边市场优势 | |

### 第三部分：优先级框架

8. **用什么框架决定优先级？**

   | 框架 | 说明 | 适用场景 |
   |------|------|---------|
   | **RICE** | Reach × Impact × Confidence / Effort | 需求量大、需要量化排序 |
   | **MoSCoW** | Must/Should/Could/Won't | 时间有限、需要砍需求 |
   | **Kano模型** | 基本型/期望型/兴奋型 | 用户体验驱动 |
   | **价值-成本矩阵** | 高价值低成本优先 | 资源有限、追求性价比 |
   | **战略对齐度** | 与战略目标的对齐程度 | 战略清晰、需要聚焦 |

9. **需求池管理规则**

   | 维度 | 规则 |
   |------|------|
   | 需求来源 | 用户反馈/数据/竞品/老板/stakeholder |
   | 需求状态 | 待评估→排队→开发→上线→关闭 |
   | 优先级 | P0(必须做)/P1(应该做)/P2(可以做)/P3(不做) |
   | 排期规则 | 谁来排？多久排一次？ |

## 虚拟角色审核：CEO视角 + 市场部视角

产品战略规划完成后，加载两个审核角色：

**CEO视角**（`references/reviewers/reviewer-ceo.md`）：
- 战略是否与公司战略对齐？
- 资源投入是否合理？
- 3年愿景够不够大？

**市场部视角**（`references/reviewers/reviewer-marketing.md`）：
- 产品定位是否清晰？
- 竞争策略是否可行？
- 路线图是否支持市场目标？

审核后输出 3-5 个关键问题和修改建议，由用户确认后再进入下一阶段。
